BSA 模块设计文档

本文档介绍用于有限尺寸标度（Finite-Size Scaling, FSS）分析的 BSA 模块的核心设计与使用方法。

物理背景

这些模块用于拟合如下形式的标度函数：

Y(X, L) = L^{c_2} × F[(X - X_c) L^{c_1}, L^{-c_3}]

其中：

  • Y：观测量（如磁化强度、关联比等）
  • X：控制参数（如温度、相互作用强度）
  • L：系统尺寸
  • X_c（= Tc 或 Uc）：临界点
  • c_1 = 1/ν：关联长度临界指数的倒数
  • c_2：反常维度标度指数
  • c_3 = ω：修正标度指数
  • F：普适标度函数

模块架构

实现分为两个互补的模块：

bsa_core.jl        ← 核心：封装外部 BSA 可执行文件
    ↓
bsa_bootstrap.jl   ← 增强：Bootstrap 重采样提供稳健误差估计

为什么需要 Bootstrap？

bsa_core.jl 虽然能快速完成单次拟合，但存在三个关键局限：

  1. 无法处理 X 的误差

    • R-dependent FSS 中，X = R 是观测量，有测量误差
    • bsa_core.jl 假设 X 精确，导致不确定性被低估

  2. 初值敏感性

    • BSA 使用高斯过程回归，对所有拟合参数（Tc, c1, c2, θ）的初值敏感
    • 单一初值容易陷入局部极小值，结果不稳健

  3. 不确定性量化不足

    • 即使开启蒙特卡洛模式（use_mc=true），也只考虑数据误差
    • 忽略了初值选择带来的拟合不确定性

Bootstrap 的解决方案

双重随机化策略：

每次 bootstrap 迭代：
  ├─ 数据随机化：X_new ~ X ± σ_X, Y_new ~ Y ± σ_Y  （处理测量误差）
  └─ 参数随机化：Tc_init ~ Tc_init ± jitter        （探索参数空间）
       └─ 多参数 jitter：同时随机化 Tc, c1, c2, ... （避免局部极小值）

最终效果：1000 次迭代 → 参数分布 → 稳健的均值 ± 标准差
